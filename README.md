# Git-Eval D ランク: Developer

チーム開発を意識した Git 運用ができるかを見るランクです。

## お題: Git の作法を意識して機能を追加しよう

既存のコードベース（または自分のプロジェクト）に対して、
Git の作法を守りながら機能を追加してください。

### やること

1. このリポジトリを clone する
2. 機能ごとにブランチを切る（例: `feature/add-login`）
3. 意味のあるコミットメッセージで commit する
4. 適切な粒度でコミットを分割する
5. push する

### 良いコミットメッセージの例

```
✅ feat: ユーザー入力のバリデーションを追加
✅ fix: ログインエラー時のメッセージを修正
✅ refactor: 認証ロジックをモジュールに分離

❌ update
❌ fix bug
❌ asdf
```

### 評価ポイント（100 点満点）

**CI/CD チェック（60 点）**

| チェック項目 | 配点 |
|---|---|
| コミットメッセージが具体的 | 20 |
| 複数コミットに分割されている | 15 |
| main 以外のブランチで作業している | 15 |
| CRLF 改行が混入していない | 10 |

**LLM 評価（40 点）**

| チェック項目 | 配点 |
|---|---|
| コミットメッセージの質（具体性・一貫性） | 20 |
| コードの分割と変更粒度の適切さ | 20 |

### 提出方法

push するだけ！ GitHub Actions が自動で評価して Discord に結果を通知します。

### GitHub Secrets の設定

| Secret 名 | 値 |
|---|---|
| `GIT_EVAL_URL` | 管理者から通知された Webhook URL |
| `GIT_EVAL_SECRET` | 管理者から通知された Secret キー |
| `ANTHROPIC_API_KEY` | Claude API キー（LLM 評価に必要） |
